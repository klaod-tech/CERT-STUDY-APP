# 자격증 학습 웹앱 개발 계획

> 정보처리기사 · TOPCIT 등 여러 자격증의 **필기 · 핵심개념 · 문제풀이**를 한 곳에서 관리하는 학습용 웹앱.
> VSCode에서 직접 구현 → GitHub Pages 배포 / 로컬 포트 사용.

---

## 1. 기술 스택

| 항목 | 선택 | 이유 |
|------|------|------|
| 프론트엔드 | **Vanilla HTML/CSS/JS** | 빌드 불필요, GitHub Pages에 파일만 올리면 즉시 동작, 디버깅 쉬움 |
| 데이터 저장 | **Supabase** (PostgreSQL + Auth) | 여러 기기 동기화, 영구 보존, 인증으로 내 데이터 격리(비공개), 무료 티어로 충분 |
| 인증 | **Supabase Auth** (이메일 로그인) | 기출 데이터를 "나만 보기"로 격리 → 저작권 리스크 해소 |
| 배포 | GitHub Pages (코드만) | 무료. **코드는 공개, 데이터는 Supabase 비공개** |

### 저작권 안전 구조 (핵심)
- **코드만 GitHub 공개**, 기출 문제 데이터는 GitHub에 올리지 않음.
- 기출 데이터는 **Supabase에 로그인한 본인 계정으로만 저장·조회** (RLS로 격리).
- Supabase 접속 키(anon key)는 공개돼도 안전하지만, **RLS(Row Level Security)** 로 "본인 데이터만 접근" 강제 → 남이 못 봄.
- 결과: 공개 저장소엔 문제 데이터가 전혀 없고, 기출은 내 계정 안에만 존재.

## 2. 화면 레이아웃 (3단 구조)

```
┌──────────┬──────────────┬────────────────────────────┐
│ 사이드바  │   기능 패널   │       메인 보드              │
│ (자격증)  │ (필기/문풀/  │  (실제 작성/문제풀이 영역)    │
│          │  핵심개념)    │                            │
│ 240px    │   220px      │       나머지 전체 (flex:1)   │
└──────────┴──────────────┴────────────────────────────┘
```

| 영역 | 너비 | 내용 |
|------|------|------|
| 사이드바 | 240px 고정 | 자격증 카테고리 목록 + 하단 `+` 버튼 |
| 기능 패널 | 220px 고정 | 선택한 자격증의 필기/문제풀이/핵심개념 탭 + "n번 진행" + 데이터 제목 리스트 |
| 메인 보드 | flex: 1 | 선택 항목 편집 영역 + 우측 하단 원형 `+` 버튼 |

### 주요 치수
- 원형 `+` 버튼: `position:absolute; bottom:24px; right:24px; width:56px; height:56px; border-radius:50%`
- 사이드바 하단 `+`: 사이드바 footer에 고정(`position:sticky; bottom:0`)
- 필기 textarea: `width:100%; height:calc(100vh - 헤더높이)`
- 핵심개념 행: `단어 input` + `뜻 input`, 각 행 높이 약 36px (메모장처럼 크지 않게)
- 리스트 스크롤: `overflow-y:auto` (세로) / 가로 카드형이면 `overflow-x:auto`

### 사이드바 접기 (Collapse)
- **트리거**: 사이드바 상단 메뉴 아이콘(☰) 클릭.
- **동작**: 사이드바(240px) + 기능패널(220px) **둘 다 접힘** → 메인보드가 전체 폭 차지.
- **접힌 상태**: 완전히 숨기지 않고 **좁은 띠(약 56px)에 아이콘만 남김** (자격증 아이콘/머리글자 등). 다시 클릭하면 펼쳐짐.
- **구현**: body 또는 최상위 컨테이너에 `.collapsed` 클래스 토글. CSS로 `grid-template-columns` 또는 너비 전환 + `transition` 으로 부드럽게.
  - 펼침: `240px 220px 1fr`
  - 접힘: `56px 0 1fr` (또는 `56px 56px 1fr`로 기능패널도 아이콘만 유지 — 기본은 기능패널 0)
- (선택) 접힘 상태 localStorage에 저장해 새로고침 후 유지.

## 3. 데이터 구조 (Supabase 테이블)

모든 테이블에 `user_id`(auth.uid()) 포함 → RLS로 본인 데이터만 접근.

```sql
-- 자격증 카테고리
categories (
  id uuid primary key default gen_random_uuid(),
  user_id uuid references auth.users not null,
  name text not null,
  created_at timestamptz default now()
)

-- 필기 (메모장형)
notes (
  id uuid primary key default gen_random_uuid(),
  user_id uuid references auth.users not null,
  category_id uuid references categories on delete cascade,
  title text,
  content text,
  created_at timestamptz default now(),
  updated_at timestamptz default now()
)

-- 핵심 개념 (단어:뜻) — items는 JSON 배열로 저장
concepts (
  id uuid primary key default gen_random_uuid(),
  user_id uuid references auth.users not null,
  category_id uuid references categories on delete cascade,
  title text,
  items jsonb default '[]',   -- [{term, meaning}, ...]
  updated_at timestamptz default now()
)

-- 문제풀이 (문제·시도기록·오답은 JSON으로 묶어 단순화)
quizzes (
  id uuid primary key default gen_random_uuid(),
  user_id uuid references auth.users not null,
  category_id uuid references categories on delete cascade,
  title text,
  source text,                 -- 'word'(단어문제) | 'exam'(기출문제)
  questions jsonb default '[]', -- [{id, question, options[4], answer}]
  attempts jsonb default '[]',  -- [{round, date, score, correct, total}]
  wrong_questions jsonb default '[]', -- [questionId, ...]
  created_at timestamptz default now()
)
```

### RLS 정책 (각 테이블 공통)
```sql
alter table categories enable row level security;
create policy "own rows" on categories
  for all using (auth.uid() = user_id) with check (auth.uid() = user_id);
-- notes, concepts, quizzes 에도 동일 적용
```
> 이 정책 덕분에 anon key가 공개돼도 **남의 데이터는 못 봄**. 각자 본인 행만 접근.

> 각 기능의 "n번 진행" 카운트는 해당 테이블 행 수 `count`로 계산.

## 4. 기능 명세

### 4-1. 사이드바 (자격증 카테고리)
- 카테고리 목록 렌더링, 클릭 시 선택 상태 강조.
- 하단 `+` 버튼 → 모달로 자격증 이름 입력 → 추가.
- (선택) 카테고리 우클릭/롱프레스 → 이름 수정·삭제.

### 4-2. 기능 패널 (필기 / 문제풀이 / 핵심개념)
- 상단 탭 3개: **필기 · 문제풀이 · 핵심개념**.
- 탭 클릭 시:
  - 상단에 "필기 n번 진행" 형태 카운트 표시.
  - 아래에 해당 데이터 **제목 리스트** 나열.
  - 리스트는 스크롤로 기존 항목 확인.
- 제목 클릭 → 메인 보드에 해당 내용 로드.

### 4-3. 필기 (메모장형)
- `title` + `textarea(content)` 구조.
- 이전 작성분 이어서 보기·수정 가능.
- **자동 저장**: `input` 이벤트 + `debounce(500ms)` → 별도 저장 버튼 불필요.
- 메인 보드 **우측 하단 원형 `+`** → 새 필기 생성.

### 4-4. 핵심 개념 (단어:뜻)
- 행 단위 입력: `[단어 input] : [뜻 input]`, 작은 칸.
- `+ 행 추가` 버튼으로 줄 늘리기.
- 자동 저장(debounce).
- 메모장보다 칸이 작아도 됨.

### 4-5. 문제풀이 (객관식 4지선다 중심)

**메인 보드 우측 상단 중앙에 토글(세그먼트): `[ 단어 문제 | 기출 문제 ]`**
두 종류를 명확히 분리한다.

**① 단어 문제 (핵심개념 자동 변환)**
   - 기존에 입력해둔 `단어:뜻` 데이터를 그대로 재활용 → 4지선다 자동 생성.
   - 정답 보기 = 해당 용어의 뜻.
   - 오답 보기 3개 = 같은 카테고리의 다른 용어 뜻 무작위 추출 후 섞기.
   - **추가 입력 불필요** (이미 핵심개념에 4개 이상 용어가 있으면 바로 출제 가능).

**② 기출 문제 (직접 데이터 입력 → 붙여넣기 파싱)**
   - 앱이 기출을 자동으로 가져오지 않음. **사용자가 데이터 공급자**.
   - 정해진 텍스트 형식으로 통째로 붙여넣으면 자동 파싱되어 문제은행에 저장.
   - 형식 예:
     ```
     1. 다음 중 OSI 7계층이 아닌 것은?
     a) 전송 계층
     b) 세션 계층
     c) 인터넷 계층
     d) 표현 계층
     정답: c
     ```
   - 줄 단위 파싱 → `questions[]` 생성 → 이후 반복 풀이 가능.

**풀이 결과 관리 (요청 4기능 전부 / 단어·기출 공통 적용)**

| 기능 | 구현 |
|------|------|
| 오답노트 자동 저장 | 채점 시 틀린 문제 id를 `wrongQuestions`에 누적 |
| 틀린 문제만 다시 풀기 | `wrongQuestions` 필터링 출제, 맞히면 배열에서 제거 |
| 회차별 기록 | 풀이마다 `attempts`에 `{round, date, score, correct, total}` 저장 |
| 정답률·점수 통계 | `attempts` 기반 회차별 정답률 추이 표시 |

**풀이 모드 흐름**
- 작성/생성 모드 ↔ 풀기 모드 토글.
- 풀기 모드: 문제 표시 → 보기 선택 → 제출 → 즉시 채점 → 결과·오답 정리.

## 4-6. 기출 데이터 수집 전략 (★ 다양한 형태 대응)

데이터가 웹복사·PDF·직접타이핑 등 **다양하게 섞이므로**, "만능 입력기"를 만들고 자동 크롤링은 하지 않는다.

### 왜 자동 크롤링을 안 하나
- CBT 사이트는 JS 동적 렌더링 + 봇 차단 → 단순 fetch 실패.
- GitHub Pages는 정적 호스팅(서버 없음) → 외부 fetch는 **CORS**로 막힘.
- 출처마다 형식이 달라 단일 파서로 자동화 불가.
- 대량 수집·공개 배포는 저작권 리스크. (→ 그래서 데이터는 Supabase 비공개)

### 채택 전략: 반자동 입력기 (난이도순)

**전략 A — 붙여넣기 변환기 (메인)**
1. 출처(웹/PDF)에서 문제 텍스트를 복사.
2. 앱의 "기출 가져오기" 모달에 붙여넣기.
3. 파서가 정해진 형식으로 분해 → 미리보기 → 확인 → Supabase 저장.
- 파싱 규칙(유연하게):
  ```
  [문제번호]. 문제 본문
  a) 보기1   (또는 ①, 1), 1. 등 다양한 마커 허용)
  b) 보기2
  c) 보기3
  d) 보기4
  정답: c   (또는 답:, 정답 3 등)
  ```
- 마커가 흔들려도 정규식으로 보기/정답을 추출하도록 관용적 파싱.
- 파싱 실패한 문제는 빨갛게 표시 → 수동 수정 후 저장.

**전략 B — PDF 일괄 변환 스크립트 (보조)**
- 시나공 등 PDF 기출이 많을 때, 로컬에서 한 번에 처리.
- 도구: Node.js `pdf-parse` 또는 Python `pdfplumber`로 텍스트 추출 → 위 형식으로 정규화 → JSON → Supabase 업로드 스크립트 실행.
- `tools/` 폴더에 분리(앱 본체와 별개, GitHub에 올려도 데이터는 없음).
- PDF가 이미지 스캔이면 OCR 필요 → 이 경우는 전략 A로 수동 처리가 빠름.

**전략 C — 자동 크롤링 → 제외**
- 차단·CORS·유지보수·저작권 부담으로 채택하지 않음.

### 출처별 가이드
| 출처 | 형태 | 방법 |
|------|------|------|
| 정보처리기사 CBT 사이트 | 웹 텍스트 | 화면 복사 → 전략 A |
| 시나공 PDF 기출 | PDF | 전략 B(많을 때) 또는 A(적을 때) |
| TOPCIT 에센스/연습문제 | 공식 자료·직접정리 | 직접 타이핑 → 전략 A |
| 직접 정리 노트 | 텍스트 | 형식 맞춰 → 전략 A |

> 모든 경로의 종착지는 동일: **파싱된 문제 → Supabase `quizzes`(본인 계정) 저장.** 코드 공개와 무관하게 데이터는 비공개 유지.

## 6. 개발 순서 (단계별)

### 0단계 — Supabase 준비
- supabase.com 프로젝트 생성 (무료 티어).
- SQL 에디터에서 위 4개 테이블 + RLS 정책 생성.
- Auth → 이메일 로그인 활성화.
- 프로젝트 URL + anon key 확보 → `supabase.js`에 기입.

### 1단계 — 기본 골격 + 인증
- `index.html` 3단 레이아웃 (CSS Grid/Flex).
- Supabase SDK 로드 + `auth.js` 로그인/로그아웃 화면.
- `store.js`: CRUD 함수 (`getCategories/addCategory/getNotes/saveNote/...`) — async/await.

### 2단계 — 사이드바
- 카테고리 목록 렌더링(Supabase에서 fetch).
- 하단 `+` → 모달 → 자격증 추가.
- 선택 상태 표시 + **상단 ☰ 접기/펼치기**(사이드바+패널 → 56px 아이콘 띠).

### 3단계 — 기능 패널
- 필기/문제풀이/핵심개념 탭 3개.
- "n번 진행" 카운트 + 제목 리스트 + 스크롤.

### 4단계 — 필기 에디터
- title + textarea, 자동 저장(debounce 500ms → Supabase update).
- 우측 하단 원형 `+` → 새 필기.

### 5단계 — 핵심개념 에디터
- `단어:뜻` 행 입력 + 행 추가, 자동 저장(items jsonb).

### 6단계 — 문제풀이
- 우측 상단 `[단어 문제 | 기출 문제]` 토글.
- 단어 문제: 핵심개념 자동 변환 출제.
- 기출 문제: **`parser.js` 붙여넣기 파서**(전략 A) → 미리보기 → 저장.
- 풀기/채점 + 오답노트 + 틀린것만 풀기 + 회차기록 + 통계.

### 7단계 — (선택) PDF 변환 도구
- `tools/pdf-to-json.js` 작성(전략 B). 필요할 때만.

### 8단계 — 마무리 & 배포
- `.gitignore`에 데이터·로컬 파일 제외 확인.

## 7. Supabase 운영 팁

- **무료 티어**: DB 500MB + 월 50,000 인증 — 개인 학습용으로 충분.
- **키 관리**: `anon key`만 프론트에 사용(공개 OK, RLS가 보호). `service_role key`는 절대 클라이언트/깃에 두지 않음.
- **백업**: Supabase 대시보드에서 주기적 export 가능 → 데이터 영구 보존.
- **여러 기기**: 같은 계정으로 로그인하면 어디서든 동일 데이터.
- **이미지가 필요해지면**: Supabase Storage 버킷 추가로 확장 가능.
